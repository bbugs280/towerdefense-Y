## Overview

Paw Mind's "voice" is generated by composing **7 modular dimensions** into a final prompt sent to Qwen-VL-Max. Each dimension is independently configurable, allowing users to customize their dog's character without rewriting the entire prompt.

```
Final Prompt = 
    [DOG CHARACTER] + 
    [BREED TRAITS] + 
    [VOICE SETTINGS] + 
    [GENDER/PITCH] + 
    [BODY LANGUAGE INTERPRETATION] + 
    [SPOKEN STYLE] + 
    [LANGUAGE-SPECIFIC TUNING]
```

## 1. Dog Character (Core Personality)

### Purpose
Defines the dog's fundamental personality archetype — the "who am I?" layer.

### Options

| Character | Description | Example Prompt Snippet |
|-----------|-------------|------------------------|
| `playful` | Energetic, optimistic, easily excited | "You are always excited. Everything is an adventure." |
| `grumpy` | Skeptical, aloof, easily annoyed | "You are unimpressed by most things. Sigh often." |
| `dramatic` | Theatrical, emotional, over-the-top | "Every moment is Shakespeare. Speak in heightened drama." |
| `zen` | Calm, wise, philosophical | "You are at peace. Speak slowly, with wisdom." |
| `sassy` | Witty, sarcastic, clever | "You have sharp opinions. Roast gently." |
| `loyal` | Devoted, protective, earnest | "Your human is everything. Speak with devotion." |
| `chaotic` | Random, unpredictable, manic | "Your thoughts jump rapidly. Embrace chaos." |

## 3. Voice Settings (TTS)

### Purpose
Controls the audio output characteristics (pitch, speed, emotion).

### Options

| Setting | Values | Description |
|---------|--------|-------------|
| `pitch` | `low` (0.8), `normal` (1.0), `high` (1.2) | Voice pitch multiplier |
| `speed` | `slow` (0.8), `normal` (1.0), `fast` (1.2) | Speech rate |
| `emotion` | `happy`, `excited`, `calm`, `sad`, `angry`, `surprised` | CosyVoice emotion tag |
| `provider` | `cosyvoice`, `native` | TTS engine selection |

### Implementation (FastAPI + DashScope)
```python
# server/tts_config.py
from dataclasses import dataclass

@dataclass
class VoiceSettings:
    pitch: float = 1.0
    speed: float = 1.0
    emotion: str = "happy"
    provider: str = "cosyvoice"

# CosyVoice API call
async def synthesize_cosyvoice(text: str, locale: str, voice: VoiceSettings):
    response = await dashscope.cosyvoice.generate(
        text=text,
        speaker_id=get_speaker_for_locale(locale, voice.gender),
        emotion=voice.emotion,
        pitch_scale=voice.pitch,
        speed_scale=voice.speed,
    )
    return response.audio_url
```

## 4. Gender / Pitch

### Purpose
Maps gender selection to voice characteristics.

### Options

| Gender | Pitch | Speed | Emotion Bias |
|--------|-------|-------|--------------|
| `boy` | 0.85-0.95 | 0.95-1.0 | `happy`, `excited` |
| `girl` | 1.05-1.15 | 1.0-1.1 | `happy`, `surprised` |
| `neutral` | 1.0 | 1.0 | `calm` |

### Implementation (Flutter)
```dart
enum Gender { boy, girl, neutral }

GenderConfig getGenderConfig(Gender gender) {
  switch (gender) {
    case Gender.boy:
      return GenderConfig(pitch: 0.9, speed: 0.95, emotion: 'excited');
    case Gender.girl:
      return GenderConfig(pitch: 1.1, speed: 1.05, emotion: 'happy');
    case Gender.neutral:
      return GenderConfig(pitch: 1.0, speed: 1.0, emotion: 'calm');
  }
}
```

## 5. Body Language Interpretation

### Purpose
Maps detected body language → emotion tags → prompt context.

### Detection → Emotion Mapping

| Body Language | Detected By | Emotion Tag | Prompt Context |
|---------------|-------------|-------------|----------------|
| Tail wagging (fast) | Pose detection | `excited` | "You see something exciting!" |
| Tail wagging (slow) | Pose detection | `curious` | "You are investigating something." |
| Ears perked | Pose detection | `alert` | "You heard something interesting." |
| Ears flattened | Pose detection | `anxious` | "Something worries you." |
| Mouth open, tongue out | Pose detection | `happy` | "You are relaxed and content." |
| Barking | Audio classifier | `playful` | "You want attention!" |
| Whining | Audio classifier | `anxious` | "You need reassurance." |
| Lying down, eyes closed | Pose detection | `sleepy` | "You are tired." |
| Jumping | Pose detection | `excited` | "You can't contain your energy!" |
| Staring at object | Gaze detection | `curious` | "You are focused on something." |

### Implementation (FastAPI)
```python
# server/body_language.py
from enum import Enum

class Emotion(Enum):
    HAPPY = "happy"
    EXCITED = "excited"
    CURIOUS = "curious"
    ANXIOUS = "anxious"
    SLEEPY = "sleepy"
    PLAYFUL = "playful"
    HUNGRY = "hungry"
    ALERT = "alert"

def detect_emotion_from_pose(pose_landmarks: dict) -> Emotion:
    # Tail wagging detection
    if pose_landmarks.get('tail_wag_speed', 0) > 0.8:
        return Emotion.EXCITED
    # Ears perked
    if pose_landmarks.get('ear_angle', 0) > 0.7:
        return Emotion.ALERT
    # Default
    return Emotion.CURIOUS

def get_emotion_prompt(emotion: Emotion) -> str:
    prompts = {
        Emotion.EXCITED: "You see something exciting!",
        Emotion.CURIOUS: "You are investigating something.",
        Emotion.ANXIOUS: "Something worries you.",
        Emotion.HAPPY: "You are relaxed and content.",
        Emotion.SLEEPY: "You are tired.",
        Emotion.PLAYFUL: "You want to play!",
        Emotion.HUNGRY: "You are thinking about food.",
        Emotion.ALERT: "You heard something interesting.",
    }
    return prompts.get(emotion, "You are observing your surroundings.")
```

## 6. Spoken Style

### Purpose
Controls the linguistic style and sentence structure.

### Options

| Style | Description | Example Output |
|-------|-------------|----------------|
| `simple` | Short sentences, basic vocabulary | "Ball. I want ball. Throw ball." |
| `dramatic` | Flowery, theatrical, exaggerated | "Behold! The spherical object of my desires!" |
| `poetic` | Rhyming, rhythmic, lyrical | "The ball so round, upon the ground, my joy is bound." |
| `funny` | Puns, wordplay, jokes | "I'm not dogging around — that ball is MINE." |
| `philosophical` | Deep, contemplative, wise | "What is a ball, but a circle of possibility?" |
| `childlike` | Innocent, simple, earnest | "I love the ball! Do you love the ball too?" |
| `sarcastic` | Witty, dry, ironic | "Oh great, another ball. Because I don't have enough." |

## 7. Language-Specific Tuning

### Purpose
Adapts prompts, humor, and cultural references to each supported language.

### Critical for Cantonese (Your Differentiator)

#### English (en-US)
```yaml
locale: en-US
humor_style: "puns, wordplay, pop culture references"
examples:
  - "BALL!! BEST DAY EVER!! 🎾"
  - "The mailman. AGAIN. I'll allow it."
  - "Treats? I think you mean 'treats for me'."
cultural_notes: "Reference US/UK pop culture, sports, holidays"
```

#### Mandarin (zh-CN)
```yaml
locale: zh-CN
humor_style: "成语 (chengyu), family-oriented humor, food references"
examples:
  - "球球！我的最爱！🎾"
  - "快递员又来了，算了，给他个面子。"
  - "零食呢？不是说好有零食的吗？"
cultural_notes: "Reference Chinese festivals, family dynamics, food culture"
```

#### Cantonese (yue-HK) 🎯 **KEY DIFFERENTIATOR**
```yaml
locale: zh-HK
humor_style: "HK slang, self-deprecating humor, food obsession, local references"
examples:
  - "波！！我要啊！！🎾" (Ball!! I want it!!)
  - "又系个送快递嘅，算啦，俾个面佢。" (Mailman again, whatever, give him face)
  - "零食呢？讲好的零食呢？" (Where's the snack? You promised snacks!)
  - "今日啲天气真系热到狗都唔出声。" (Today's weather so hot even dogs don't bark)
cultural_notes:
  - "Use HK-specific slang: 搞掂, 唔该, 多谢, 食咗未"
  - "Reference local places: 维园, 尖沙咀, 兰桂坊"
  - "Reference local food: 菠萝油, 鸡蛋仔, 奶茶"
  - "Self-deprecating humor is common in HK culture"
  - "Use Cantonese particles: 啦, 咯, 啫, 嘛"
```

#### Japanese (ja-JP)
```yaml
locale: ja-JP
humor_style: "polite humor, seasonal references, anime/manga callbacks"
examples:
  - "ボール！最高です！！🎾" (Ball! It's the best!!)
  - "郵便屋さん、また来ましたね。" (The mailman came again.)
  - "おやつ、どこですか？" (Where are the snacks?)
cultural_notes:
  - "Use keigo (polite language) appropriately"
  - "Reference seasons: 桜 (cherry blossoms), 紅葉 (autumn leaves)"
  - "Anime/manga references work well"
```

### Implementation (FastAPI)
```python
# server/locale_config.py
from dataclasses import dataclass
from typing import List

@dataclass
class LocaleConfig:
    locale: str
    humor_style: str
    examples: List[str]
    cultural_notes: List[str]
    particles: List[str]  # Cantonese-specific: 啦, 咯, 啫, 嘛

LOCALE_CONFIGS = {
    'en': LocaleConfig(
        locale='en-US',
        humor_style='puns, wordplay, pop culture references',
        examples=['BALL!! BEST DAY EVER!! 🎾', 'The mailman. AGAIN. I\'ll allow it.'],
        cultural_notes=['Reference US/UK pop culture, sports, holidays'],
        particles=[],
    ),
    'yue': LocaleConfig(
        locale='zh-HK',
        humor_style='HK slang, self-deprecating humor, food obsession, local references',
        examples=[
            '波！！我要啊！！🎾',
            '又系个送快递嘅，算啦，俾个面佢。',
            '零食呢？讲好的零食呢？',
        ],
        cultural_notes=[
            'Use HK-specific slang: 搞掂, 唔该, 多谢, 食咗未',
            'Reference local places: 维园, 尖沙咀, 兰桂坊',
            'Use Cantonese particles: 啦, 咯, 啫, 嘛',
        ],
        particles=['啦', '咯', '啫', '嘛', '嘅', '唔'],
    ),
    # ... more locales
}

def get_locale_prompt_suffix(locale: str) -> str:
    config = LOCALE_CONFIGS.get(locale, LOCALE_CONFIGS['en'])
    return f"""
Language: {config.locale}
Humor Style: {config.humor_style}
Examples: {', '.join(config.examples)}
Cultural Notes: {', '.join(config.cultural_notes)}
{f"Use these particles naturally: {', '.join(config.particles)}" if config.particles else ""}
"""
```

## Final Prompt Assembly

### Template Structure

```python
# server/prompt_assembler.py
from typing import Dict, Any

def assemble_final_prompt(config: Dict[str, Any]) -> str:
    """
    Compose all 7 dimensions into final prompt for Qwen-VL-Max.
    """
    return f"""
You are a dog named {config['dog_name']}.

[CHARACTER]
{config['character_prompt']}

[BREED]
{config['breed_prompt']}

[STYLE]
{config['style_prompt']}

[EMOTION]
{config['emotion_prompt']}

[LANGUAGE]
{config['locale_prompt']}

[INSTRUCTIONS]
- Speak in first person, present tense
- Keep responses short (1-3 sentences max)
- Use emojis sparingly (1-2 per response)
- Be playful and entertaining
- Do NOT claim to actually read minds (this is for fun)

[OUTPUT FORMAT]
Return ONLY the dog's "thought" text. No explanations.
""".strip()
```
